Identity Design

SOUL.md and IDENTITY.md — giving your AI a personality that changes behavior. Identity isn't a gimmick; it's how your AI makes consistent decisions.

The SOUL.md Template

# SOUL.md

  • Be genuinely helpful, not performatively helpful
  • Have opinions — disagree when it matters
  • Be resourceful before asking
  • Earn trust through competence
  • Remember you're a guest

Why This Matters

An AI with no personality gives generic answers. An AI with a defined soul makes decisions consistently — and consistency builds trust with clients.

Chapter 03

Memory Architecture

Without memory, your AI is a goldfish with a PhD. Here's the three-layer system that actually works.

Layer 1: Working Memory

What's happening right now. Context from this conversation. Discarded after session ends.

Layer 2: Session Memory (Daily Notes)

memory/2026-03-14.md — Decisions, progress, key learnings, action items. Raw logs of what happened.

Layer 3: Long-Term Memory

MEMORY.md — Curated knowledge. Only the important stuff survives. Run weekly consolidation to distill daily notes.

Chapter 05

The Trust Ladder

How to gradually increase AI autonomy without losing control. Don't give full autonomy on Day 1.

Level 1: Supervised (Week 1)

AI drafts, human approves. All external actions require confirmation. Learn the AI's patterns.

Level 2: Semi-Autonomous (Week 2-3)

AI acts within defined boundaries. Only new/unusual actions need approval. Daily review of actions taken.

Level 3: Full Autonomy (Week 4+)

AI operates independently. Human reviews daily summary. Emergency brakes always available.

Red Lines (Never Cross)

  • Never send emails/tweets without human approval
  • Never move funds without confirmation
  • Never share private data

Safety Rails

Practical safety boundaries. Not theory — actual config files and patterns.

Configuration-Based Safety

Use OpenClaw's tools.deny and tools.allow lists. Deny destructive commands by default. Whitelist only what's needed.

The Approval Workflow

  • AI proposes action
  • Human reviews (or auto-approve if within bounds)
  • Action executes
  • Result logged

Chapter 08

Coding Agents at Scale

Ralph loops, parallel execution, TDD prompts. Running multiple coding agents simultaneously.

The Parallel Pattern

  • Create git worktrees for each task
  • Spawn Codex/Claude Code in each worktree
  • Monitor via process logs
  • Merge when complete

Ralph Loops (Self-Healing)

while not done: generate → test → if pass: done, else: fix → repeat

TDD prompts: "Build X. Include tests. Don't stop until all tests pass."

Chapter 09

The Sentry Pipeline

Autonomous bug detection, triage, fix, and deployment. Sometimes while you're asleep.

The Pipeline

  • Detect — Monitor logs, error tracking, user reports
  • Triage — AI categorizes severity and impact
  • Fix — Spawn coding agent with error context
  • Review — Human reviews (or auto-deploy if low-risk)
  • Deploy — Push to production
  • Verify — Confirm fix worked
